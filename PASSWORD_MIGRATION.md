# 密码迁移说明

## 问题原因

项目将密码加密方式从 **SHA-256** 升级为 **BCrypt**，导致之前注册的用户无法登录。

## 解决方案

已创建 `PasswordMigrationRunner` 自动迁移脚本，会在应用启动时自动执行：

### 迁移逻辑

1. 检查每个用户的密码格式
2. 如果密码是 BCrypt 格式（以 `$2a$` 开头）→ 跳过，保持不变
3. 如果密码是旧的 SHA-256 格式（64位十六进制）→ 重置为默认密码 `123456`

### 默认账号（DataInitializer 创建）

| 用户名 | 密码 | 角色 | 说明 |
|--------|------|------|------|
| admin | 123456 | admin | 管理员账号 |
| test | 123456 | user | 测试用户 |

### 如何判断迁移是否执行

启动应用后，查看控制台日志：

```
迁移用户 'admin' 的密码到 BCrypt 格式
用户 'admin' 密码已重置为默认密码 '123456'，请提醒用户登录后修改密码
迁移用户 'test' 的密码到 BCrypt 格式
用户 'test' 密码已重置为默认密码 '123456'，请提醒用户登录后修改密码
密码迁移检查完成
```

## 关于旧密码

⚠️ **重要说明**：
- SHA-256 是单向哈希函数，**无法逆向解密**获取原始密码
- 因此迁移时**无法保留用户原来的密码**
- 所有使用旧密码格式的账号密码都会被重置为 `123456`

## 如果需要保留某些特定用户的密码

如果有需要保留密码的用户（如管理员账号），可以手动更新数据库：

```sql
-- 手动设置 BCrypt 格式的密码（这里是 "admin123" 的 BCrypt 哈希）
UPDATE users 
SET password = '$2a$10$...（BCrypt哈希值）...'
WHERE username = 'admin';
```

可以使用在线工具生成 BCrypt 哈希：
- https://bcrypt-generator.com/
- https://www.browserling.com/tools/bcrypt

## BCrypt vs SHA-256 的优势

1. **自带盐值**：每个密码都有独特的盐，无需额外处理
2. **可配置强度**：可以调整工作因子（cost factor）
3. **自适应**：随着硬件发展可以增加计算成本，防止暴力破解
4. **专为密码设计**：包含防时序攻击的比较算法

## 测试步骤

1. 启动应用
2. 查看日志确认迁移完成
3. 使用以下账号登录：
   - 用户名：`admin`，密码：`123456`
   - 用户名：`test`，密码：`123456`
4. 登录后建议修改密码
